## モンスターの種類

モンスター タイプは、"type" メンバーが "MONSTER" に設定された JSON オブジェクトとして指定されます。

```json
{
    "type": "MONSTER",
    "id": "mon_foo",
    ...
}
```

id メンバーは、その型の一意の ID である必要があります。任意の文字列にすることができますが、慣例により、
プレフィックス "mon_"。この ID は、モンスター グループや MapGen など、さまざまな場所で参照できます。
特定のモンスターを出現させます。

数量文字列 (体積、重量など) には、完全な精度を維持できる最大の単位を使用します。

モンスター タイプは次のプロパティをサポートしています (特に明記されている場合を除き、必須)。

## "name"

(文字列またはオブジェクト)

```json
"name": "cow"
```

```json
"name": { "ctxt": "fish", "str": "pike", "str_pl": "pikes" }
```

または、単数形と複数形が同じ場合:

```json
"name": { "ctxt": "fish", "str_sp": "bass" }
```

ゲーム内に表示される名前、およびオプションで複数名と翻訳コンテキスト (ctxt)。

複数名が指定されていない場合は、デフォルトで単数名 + "s" が使用されます。

Ctxt は、同音異義語 (同じ名前を持つ 2 つの異なるもの) の場合に翻訳者を支援するために使用されます。のために
たとえば、魚を刺し、武器を刺します。

## "lua_attitude"

(文字列、オプション)

設定されている場合、`game.monster_attitude_functions` の Lua 関数を使用してモンスターの属性を決定します。
デフォルトの C++ ロジックの代わりに態度を使用します。 Lua 関数は `(monster, target)` で呼び出されます。
`target` は `nil` にすることができ、`MonsterAttitude` 値を返す必要があります。 `nil` を返すとフォールバックします
デフォルトの動作に戻ります。

```json
"lua_attitude": "my_attitude"
```

```lua
game.monster_attitude_functions["my_attitude"] = function(mon, target)
  if target ~= nil and target:is_avatar() then
    return MonsterAttitude.MATT_ATTACK
  end
  return nil
end
```

## "lua_ai"

(文字列、オプション)

設定されている場合、各モンスター ターン `game.monster_ai_functions` から Lua 関数が実行されます。機能
`(monster)` で呼び出され、ブール値を返す必要があります: `true` は Lua がターンを処理したことを意味し、
デフォルトの C++ AI はスキップされ、`false` または `nil` はデフォルトの AI に戻ります。

```json
"lua_ai": "my_ai"
```

```lua
game.monster_ai_functions["my_ai"] = function(mon)
  local avatar = gapi.get_avatar()
  if avatar ~= nil then
    local function sign(val)
      if val > 0 then
        return 1
      end
      if val < 0 then
        return -1
      end
      return 0
    end
    local pos = mon:get_pos_ms()
    local tgt = avatar:get_pos_ms()
    local step = Tripoint.new(pos.x + sign(tgt.x - pos.x), pos.y + sign(tgt.y - pos.y), pos.z)
    mon:set_target(avatar)
    mon:move_to(step, false, true, 1.0)
  end
  return true
end
```

## "monster_weapon"

(項目グループ、オプション)

モンスターが使用する武器と思われるアイテムを生成するために使用されるアイテム グループ。これを定義すると、モンスターが武装解除テクニックに対して脆弱になります。武装解除された場合、モンスターはバッシング ダメージ ロールが大幅に減少し、特殊近接ダメージ (カット、電気など) がゼロになり、特定の特殊攻撃 (特に `"type": "gun"` の攻撃だけでなく、`TAZER`、`SHOCKSTORM`、`FLAMETHROWER` などの一部の攻撃) を使用できなくなります。

テクニックが `take_weapon` に設定されている場合でも、アイテムは攻撃者が使用するのではなく地面にドロップされるため、両手で使用する 2 つの武器を暗示するコレクションを自由に使用できます。モンスターが武装解除されずに殺された場合、アイテムグループは `death_drops` と同様に死亡時にも生成されます。

## "death_drops"

(文字列または項目グループ、オプション)

モンスターが死亡したときにアイテムを生成するために使用されるアイテム グループ。これはインライン項目グループにすることができます。
ITEM_SPAWN.md を参照してください。デフォルトのサブタイプは "distribution" です。

継承またはオーバーライドされたモンスターで `"extend": { "death_drops": ... }` を使用すると、既存のデス ドロップを置き換えずに別のドロップ グループを追加できます。最上位の `"death_drops"` メンバーは、継承された死亡ドロップを置き換えます。

## "special_attacks"

(特別な攻撃定義の配列、オプション)

モンスターの特殊攻撃。これは配列である必要があり、その各要素はオブジェクトである必要があります (新しい
スタイル) または配列 (古いスタイル)。

古いスタイルの配列には、攻撃の ID の 2 つの要素が含まれている必要があります (リストについては、json_flags.md を参照してください)。
そしてその攻撃のクールダウン。例 (10 ターンごとに攻撃を掴む):

```json
"special_attacks": [ [ "GRAB", 10 ] ]
```

新しいスタイル オブジェクトには、少なくとも "type" メンバー (文字列) と "cooldown" メンバーが含まれている必要があります。
(整数)。特定の型の必要に応じて追加のメンバーが含まれる場合があります。考えられるタイプは次のとおりです
以下に列挙します。例:

```json
"special_attacks": [
    { "type": "leap", "cooldown": 10, "max_range": 4 }
]
```

"special_attacks" には、古いスタイルのエントリと新しいスタイルのエントリが混在している場合があります。

```json
"special_attacks": [
    [ "GRAB", 10 ],
    { "type": "leap", "cooldown": 10, "max_range": 4 }
```

# 改造

モンスターのタイプは MOD で上書きまたは変更できます。そのためには、"edit-mode" メンバーを追加する必要があります。
次のいずれかを含めることができます。

- "create" (メンバーが存在しない場合のデフォルト)、タイプが次の場合はエラーが表示されます。
  指定された ID はすでに存在します。
- "override"、指定された ID を持つ既存のタイプ (存在する場合) が削除され、新しいデータが作成されます。
  完全に新しいタイプとしてロードされます。
- "modify"、既存のタイプが変更されます。指定された ID のタイプがない場合は、エラーが発生します。
  示される。

必須プロパティ (オプションとしてマークされていないすべて) は、編集モードが の場合にのみ必要です。
"create" または "override"。

例 (ゾンビ モンスターの名前を変更し、他のすべてのプロパティは変更しない):

```json
{
  "type": "MONSTER",
  "edit-mode": "modify",
  "id": "mon_zombie",
  "name": "clown"
}
```

デフォルトの編集モード ("create") は、新しいタイプの ID がタイプと競合する場合に適しています。
他のMODやコアデータからの場合、エラーが表示されます。編集モード "override" が適しています
型を最初から再定義する場合、すべての必須メンバーがリストされ、何も残らないことが保証されます。
以前の定義の痕跡。編集モード "modify" は、フラグの追加や
特殊攻撃を解除します。

タイプを変更すると、プロパティが新しい値でオーバーライドされます。この例では、特別な攻撃を設定します。
"SHRIEK" 攻撃_のみ_ を封じ込めるには:

```json
{
  "type": "MONSTER",
  "edit-mode": "modify",
  "id": "mon_zombie",
  "special_attacks": [["SHRIEK", 20]]
}
```

一部のプロパティでは、上記で説明したように、通常は、
"add:"/"remove:" プレフィックス。
